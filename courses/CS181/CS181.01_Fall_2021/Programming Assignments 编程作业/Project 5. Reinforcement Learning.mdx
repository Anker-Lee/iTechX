<p style="box-sizing: border-box; margin: 0.5rem 0px; line-height: 1.5em; color: #212529; font-family: Lato; font-size: 15.2px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-style: initial; text-decoration-color: initial;"><em style="box-sizing: border-box;">Important:</em> Use the “batch” version of value iteration where each vector $ V_k$ is computed from a fixed vector $ V_{k-1}$ (like in lecture), not the “online” version where one single weight vector is updated in place. This means that when a state's value is updated in iteration $ k$ based on the values of its successor states, the successor state values used in the value update computation should be those from iteration $ k-1$ (even if some of the successor states had already been updated in iteration $ k$). Here is how cyclic value iteration works. In the first iteration, only update the value of the first state in the states list. In the second iteration, only update the value of the second. Keep going until you have updated the value of each state once, then start back at the first state for the subsequent iteration. <strong style="box-sizing: border-box; font-weight: bolder;">If the state picked for updating is terminal, nothing happens in that iteration</strong>. You can implement it as indexing into the states variable defined in the code skeleton.</p>
<p style="box-sizing: border-box; margin: 0.5rem 0px; line-height: 1.5em; color: #212529; font-family: Lato; font-size: 15.2px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-thickness: initial; text-decoration-style: initial; text-decoration-color: initial;">As a reminder, here's the value iteration state update equation:</p>
<center>$ V_{k+1}(s) \leftarrow \max _{a} \sum_{s^{\prime}} T\left(s, a, s^{\prime}\right)\left[R\left(s, a, s^{\prime}\right)+\gamma V_{k}\left(s^{\prime}\right)\right]$</center>
<p style="box-sizing: border-box; margin: 0.5rem 0px; line-height: 1.5em; color: #212529; font-family: Lato; font-size: 15.2px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-thickness: initial; text-decoration-style: initial; text-decoration-color: initial;">Value iteration iterates a fixed-point equation, as discussed in class. Write your implementation in <code style="box-sizing: border-box; font-family: Inconsolata, Menlo, Monaco, 'Courier New', Courier, monospace; font-size: 14.44px; color: #1e90ff; word-break: break-word; white-space: nowrap; background-color: #f0f7ff; padding: 1px 4px; border-radius: 5px; border: 1px solid rgba(30, 144, 255, 0.5);" class="highlighter-rouge">ApproximateQAgent</code> class in <code style="box-sizing: border-box; font-family: Inconsolata, Menlo, Monaco, 'Courier New', Courier, monospace; font-size: 14.44px; color: #1e90ff; word-break: break-word; white-space: nowrap; background-color: #f0f7ff; padding: 1px 4px; border-radius: 5px; border: 1px solid rgba(30, 144, 255, 0.5);" class="highlighter-rouge">qlearningAgents.py</code>, which is a subclass of <code style="box-sizing: border-box; font-family: Inconsolata, Menlo, Monaco, 'Courier New', Courier, monospace; font-size: 14.44px; color: #1e90ff; word-break: break-word; white-space: nowrap; background-color: #f0f7ff; padding: 1px 4px; border-radius: 5px; border: 1px solid rgba(30, 144, 255, 0.5);" class="highlighter-rouge">PacmanQAgent</code>.</p>
<p style="box-sizing: border-box; margin: 0.5rem 0px; line-height: 1.5em; color: #212529; font-family: Lato; font-size: 15.2px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-style: initial; text-decoration-color: initial;"><em style="box-sizing: border-box;">Note:</em> Approximate Q-learning assumes the existence of a feature function $ f(s,a)$ over state and action pairs, which yields a vector $ [f_1(s,a), \dots, \ f_i(s,a), \dots, \ f_n(s,a)]$ of feature values. <p style="box-sizing: border-box; margin: 0.5rem 0px; line-height: 1.5em; color: #212529; font-family: Lato; font-size: 15.2px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-style: initial; text-decoration-color: initial;">The approximate Q-function takes the following form:</p>
<center>$ Q(s, a)=\sum_{i=1}^{n} f_{i}(s, a) w_{i}$</center>
<p style="box-sizing: border-box; margin: 0.5rem 0px; line-height: 1.5em; color: #212529; font-family: Lato; font-size: 15.2px; font-style: normal; font-variant-ligatures: normal; font-variant-caps: normal; font-weight: 400; letter-spacing: normal; orphans: 2; text-align: left; text-indent: 0px; text-transform: none; white-space: normal; widows: 2; word-spacing: 0px; -webkit-text-stroke-width: 0px; text-decoration-style: initial; text-decoration-color: initial;">where each weight $ w_i$ is associated with a particular feature $ f_i(s,a)$. <center>$ \begin{array}{c}{w_{i} \leftarrow w_{i}+\alpha \cdot \text {difference} \cdot f_{i}(s, a)} \\ {\text {difference}=\left(r+\gamma \max _{a^{\prime}} Q\left(s^{\prime}, a^{\prime}\right)\right)-Q(s, a)}\end{array}$</center>
